# Folder Structure for v3.0 Architecture

This document outlines the folder structure for the v3.0 architecture of the ECG Digital City project.

## Root Directory

- **/app**  
  Contains all application-related code and resources.
  
  - **/controllers**  
    Contains controller classes to handle user requests and responses.
  
  - **/models**  
    Contains data models representing the core data structures of the application.
  
  - **/views**  
    Contains templates and static files used for rendering views.
  
- **/config**  
  Contains configuration files to manage application settings.
  
- **/assets**  
  Contains all assets including CSS, JavaScript, images, and fonts.
  
- **/tests**  
  Contains test scripts and case files to ensure code reliability.
  
- **/docs**  
  Contains documentation files including this folder structure documentation.

## Special Directories

- **/scripts**  
  Contains scripts for automation, deployment, and other utility functions.  
  
- **/migrations**  
  Contains database migration files to handle schema changes smoothly.
  
- **/logs**  
  Contains log files for debugging and monitoring application runtime behavior.

## Notes

Ensure to follow the established naming conventions and folder structure for better maintainability and scalability of the application.
